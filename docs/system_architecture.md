# System Architecture - Integrated ERP Workflow

## Database Schema Documentation

### 1. Projects Table
**Table:** `projects`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PRIMARY KEY | Unique project identifier |
| quotation_id | INTEGER | NOT NULL, FK → quotations(id) | Linked quotation |
| client_id | INTEGER | FK → users(id) | Client user reference |
| name | VARCHAR(255) | NOT NULL | Project name |
| description | TEXT | | Project description |
| budget | NUMERIC(16,2) | DEFAULT 0 | Project budget |
| start_date | DATE | | Project start date |
| end_date | DATE | | Project end date |
| status | VARCHAR(50) | DEFAULT 'planning' | Project status |
| **contract_status** | **VARCHAR(20)** | **DEFAULT 'not_uploaded'** | **Contract upload status** |
| created_at | TIMESTAMPTZ | DEFAULT now() | Creation timestamp |
| updated_at | TIMESTAMPTZ | DEFAULT now() | Last update timestamp |

**contract_status Values:**
- `not_uploaded` - No contract uploaded yet
- `uploaded` - Contract file uploaded
- `verified` - Contract reviewed and verified

**Foreign Keys:**
- quotation_id → quotations(id) ON UPDATE CASCADE ON DELETE SET NULL
- client_id → users(id) ON UPDATE CASCADE ON DELETE SET NULL

### 2. Contracts Table
**Table:** `contracts`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PRIMARY KEY | Unique contract identifier |
| contract_number | VARCHAR(100) | UNIQUE, NOT NULL | Contract reference number |
| project_id | INTEGER | NOT NULL, FK → projects(id) | Linked project |
| client_id | INTEGER | NOT NULL, FK → users(id) | Client reference |
| contract_type | VARCHAR(50) | DEFAULT 'service' | Type: service, maintenance, supply |
| start_date | DATE | NOT NULL | Contract start date |
| end_date | DATE | NOT NULL | Contract end date |
| total_value | DECIMAL(15,2) | NOT NULL | Contract total value |
| currency | VARCHAR(10) | DEFAULT 'SAR' | Currency code |
| payment_terms | TEXT | | JSON payment milestones |
| description | TEXT | | Contract description |
| **attachment_url** | **VARCHAR(500)** | | **PDF file path** |
| status | VARCHAR(30) | DEFAULT 'active' | active, expired, terminated, pending_signature |
| signed_by_client | BOOLEAN | DEFAULT FALSE | Client signature status |
| signed_by_company | BOOLEAN | DEFAULT FALSE | Company signature status |
| signed_at | TIMESTAMPTZ | | Signature timestamp |
| created_by | INTEGER | FK → users(id) | Creator user |
| created_at | TIMESTAMPTZ | DEFAULT now() | Creation timestamp |
| updated_at | TIMESTAMPTZ | DEFAULT now() | Last update timestamp |

**Foreign Keys:**
- project_id → projects(id) ON DELETE CASCADE
- client_id → users(id) ON DELETE CASCADE
- created_by → users(id) ON DELETE SET NULL

**File Storage:**
- Path: `uploads/contracts/`
- Format: PDF files
- Naming: Auto-generated by multer

### 3. Inventory Items Table
**Table:** `inventory_items`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PRIMARY KEY | Unique item identifier |
| item_code | VARCHAR(50) | UNIQUE, NOT NULL | Item code (e.g., ITM-0001) |
| item_name | VARCHAR(255) | NOT NULL | Item name |
| item_name_ar | VARCHAR(255) | | Arabic item name |
| category | VARCHAR(50) | NOT NULL | solar_panel, inverter, cable, accessory, other |
| unit_of_measure | VARCHAR(20) | DEFAULT 'pcs' | pcs, m, kg, set, box, roll |
| coa_account_code | VARCHAR(20) | NOT NULL | Chart of Accounts code |
| cost_account_code | VARCHAR(20) | NOT NULL | Cost account code |
| unit_cost | NUMERIC(15,2) | DEFAULT 0 | Cost per unit |
| **quantity_on_hand** | **NUMERIC(15,3)** | **DEFAULT 0** | **Available stock** |
| **reorder_level** | **NUMERIC(15,3)** | **DEFAULT 0** | **Minimum stock threshold** |
| is_active | BOOLEAN | DEFAULT true | Item active status |
| notes | TEXT | | Additional notes |
| created_by | INTEGER | FK → users(id) | Creator |
| created_at | TIMESTAMPTZ | DEFAULT NOW() | Creation timestamp |
| updated_at | TIMESTAMPTZ | DEFAULT NOW() | Last update timestamp |

**Low Stock Condition:**
```sql
quantity_on_hand <= reorder_level
```

### 5. Inventory Movements (Project Materials)
**Table:** `inventory_movements`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PRIMARY KEY | Unique movement identifier |
| inventory_item_id | INTEGER | NOT NULL, FK → inventory_items(id) | Item reference |
| project_id | INTEGER | FK → projects(id) | Project reference |
| movement_type | VARCHAR(20) | NOT NULL | in, out, transfer |
| quantity | INTEGER | NOT NULL, CHECK > 0 | Movement quantity |
| performed_by | INTEGER | FK → users(id) | User who performed movement |
| performed_at | TIMESTAMPTZ | DEFAULT now() | Movement timestamp |
| notes | TEXT | | Movement notes |
| created_at | TIMESTAMPTZ | DEFAULT now() | Creation timestamp |
| updated_at | TIMESTAMPTZ | DEFAULT now() | Last update timestamp |

**Movement Types:**
- `in` - Stock received/increased
- `out` - Stock allocated/decreased (project material allocation)
- `transfer` - Stock transferred between locations

## API Endpoints Summary

### Contract Management
- `POST /api/contracts/:projectId/upload` - Upload contract file
- `GET /api/contracts/:id` - Get contract details
- `GET /api/projects/:id/contracts` - Get project contracts

### Inventory Management
- `GET /api/inventory/items` - List all inventory items
- `GET /api/inventory/categories` - Get distinct categories (DYNAMIC)
- `GET /api/inventory/items/:id` - Get item details
- `PATCH /api/inventory/items/:id` - Update item

### Material Allocation
- `POST /api/projects/:id/materials` - Allocate materials to project
- `GET /api/projects/:id/materials` - Get allocated materials

### Purchase Requests
- `POST /api/purchase-requests` - Create purchase request
- `GET /api/purchase-requests` - List all requests
- `PATCH /api/purchase-requests/:id/status` - Update request status

## Business Rules

1. **Contract-Driven Resource Locking:**
   - PM cannot allocate materials unless `project.contract_status = 'uploaded'`
   - Only `contract_dept_head` can upload contracts
   - After upload, redirect to Project Reports page

2. **Stock Validation:**
   - If `requested_quantity > quantity_on_hand`, show "Request Purchase" button
   - Low stock alert when `quantity_on_hand <= reorder_level`

3. **Notifications:**
   - Material allocation → Notify Warehouse Manager + Finance Manager
   - Purchase request → Notify Warehouse Manager + Finance Manager

4. **Dynamic Categories:**
   - Fetch categories from database: `SELECT DISTINCT category FROM inventory_items`
   - No static category filters in frontend
